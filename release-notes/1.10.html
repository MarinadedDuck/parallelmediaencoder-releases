What's New in 1.10

Watch folder hardening — network share credentials, smart error handling, drag-and-drop folder setup, configurable PickedUp destinations, and full scan logging.

  • New: Network Share Credentials (Pro) — Store username and password per watch folder for automatic remounting when SMB/NFS/AFP shares disconnect. Passwords stored securely in macOS Keychain
  • New: Auto-Remount — PME detects disconnected network shares and automatically attempts to remount using stored credentials. Manual "Mount" button available in the watch folder row
  • New: Watch Folder Validation — Output folder cannot be inside the watch folder (prevents infinite loops). Watch folders cannot overlap as parent/child of each other
  • New: Custom PickedUp Destinations — Move detected files to any folder on the same volume instead of the default PickedUp subfolder. Configurable PickedUp folder name in Settings
  • New: Drag & Drop Folder Setup — Drag folders from Finder directly onto the Watch Folder, Output, and PickedUp fields in the settings sheet
  • New: Watch Folder Scan Logging — Full scan activity logged to the console window with file counts, stability checks, move operations, and error diagnosis
  • New: Expanded Insta360 Metadata — Improved metadata extraction for Insta360 .insv footage
  • Improved: Structured Error Handling — Watch folder errors now distinguish between folder deleted, volume unmounted, network disconnected, and permission denied
  • Improved: Clickable Folder Pills — Watch, Output, and Moved-to folder names shown as colored pills that open the folder in Finder
  • Improved: CLI — New --picked-up-path, --network-user, --network-password flags for pme watch add
