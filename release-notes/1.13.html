What's New in 1.13

ProRes RAW colour profiles, native ProRes-in-MXF decode, major queue and preset UI improvements, and critical bug fixes for watermark/trim export, DNxHD output, and queue selection.

  • New: ProRes RAW Colour Profiles — ProRes RAW sources now support Flat and Rec.709 colour profiles with a redesigned colour pipeline
  • New: ProRes-in-MXF Decode — Natively decode Apple ProRes (422 / 422 HQ / 4444 / 4444 XQ) inside MXF containers via VideoToolbox. Preview, scrub, and transcode to all output codecs (HEVC, H.264, ProRes, DNxHR, DNxHD) with full audio passthrough. No extra plugins or SDKs required
  • New: DNxHD CLI Codec Mappings — pme transcode --codec dnxhd-lb/sq/hq/hqx/444 now available alongside the existing DNxHR variants
  • New: Job Status Banners — The settings panel shows a live-updating banner for queued, encoding, and completed jobs. Completed jobs have a clickable banner that opens the output file in Finder
  • New: Reset Status — Right-click completed, failed, or cancelled jobs in the queue to re-run them
  • New: Encoder Stats in Queue — Optional "Show encoder FPS and realtime rate" toggle in Settings → General → Queue displays live encoding speed on running jobs
  • New: Preset Detail View — Expand any preset with the chevron icon to see all saved settings at a glance
  • Improved: "Applied" Preset Feedback — The Apply Preset button shows "Applied" when the selected preset matches the current job settings, and reverts when you make changes
  • Improved: Reactive Settings Editability — Settings lock and unlock in real time as jobs transition between queued, encoding, and completed states
  • Fixed: Watermark Export Regression — Watermarks now appear in encoded output when "Keep sources in staging" is enabled. Previously, watermark settings were lost when jobs entered the queue. Trim ranges, GIF loop count, and several other settings were also affected and are now fixed
  • Fixed: DNxHD Output — DNxHD codec variants now correctly produce DNxHD streams instead of silently falling back to DNxHR
  • Fixed: Queue Selection Highlight — Row highlights in the queue panel no longer freeze after encoding finishes
  • Fixed: Preset Tag on No-Audio Sources — Preset names no longer disappear on import for sources without audio tracks
  • Fixed: ⌘A in Filename Field — ⌘A now selects text in the filename field instead of selecting all sources. It also selects only the basename, preserving the file extension
  • Fixed: Insta360 Compatibility Message — ProRes and DNx codec family warnings now show the correct family name instead of the specific variant
