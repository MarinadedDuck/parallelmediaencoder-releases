What's New in 1.123

Audio settings cleanup, smarter Automatic mode, file size estimation for all codecs, HDR overlay fixes, and a clearer update permission flow.

  • New: Simplified Audio Settings — Single source of truth for track handling with clearer labels: Keep All Tracks, Mixdown to Mono, Mixdown to Stereo
  • New: Automatic Mode Reset — Reset button beside Automatic re-evaluates audio and codec defaults from current source metadata
  • New: Automatic Codec Defaults — HEVC auto-selects Main/Main10 based on source bit depth; H.264 auto-selects High profile
  • New: File Size Estimation for All Codecs — The estimation table now appears for ProRes, DNxHR, DNxHD, and GIF — not just H.264/HEVC. Uses Apple-published ProRes data rates and matched DNx rates scaled by resolution and frame rate
  • New: Update Permission Screenshot — The App Management permission alert now includes a screenshot showing exactly which toggle to enable
  • Improved: No-Audio Handling — Sources without audio auto-select No Audio. Batch settings skip no-audio clips. No Audio added as explicit batch option
  • Improved: Batch Audio — Batch audio changes apply to all selected clips and pending probes
  • Improved: Automatic Audio — Compares pass-through vs re-encode size and picks the smaller output
  • Fixed: HDR pass-through on rotated clips — HDR sources (e.g. BT.2020 HLG from iPhone) no longer appear burnt/orange after rotation or scaling
  • Fixed: HDR with overlays — Timecode/watermark overlays no longer break HDR pass-through
  • Fixed: Overlay preview accuracy — Timecode and watermark placement in preview now matches final output across vertical and 16:9 sources
  • Fixed: Timecode size slider — Low-end values are no longer a dead zone
  • Fixed: Conflicting audio options, preset-on-import no-audio edge case, Automatic timing stability
