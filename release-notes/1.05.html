What's New in 1.05

  • New: Conflict resolution sheet — when queueing files with naming conflicts, a new sheet shows all conflicts with editable filenames, real-time validation, and per-item remove buttons
  • New: Smart numeric suffixes — output collisions are resolved with _1, _2, etc. Re-queueing video_1.mov correctly increments to _2 instead of _1_1
  • New: Queue now shows the output filename instead of the source filename
  • New: You can now overwrite existing output files — the app warns and offers Replace, Rename, or Go Back instead of forcing a rename
  • Improved: Sources selection highlight for easier identification
  • Improved: Update checks on every launch
  • Improved: Audio codec validation — bit depth clamped to valid ranges for the selected codec
  • Improved: Output suffix text field is always editable
  • Fixed: Overlay and LUT settings lost when "Keep sources in staging" was enabled
  • Fixed: Burn-in timecode drift — frame counter now uses rounding instead of truncation
  • Fixed: Burn-in timecode mismatch when output frame rate differs from source
  • Fixed: Overlay scaling when output resolution differs from source
  • Fixed: AAC audio encoding failure with 32-bit bit depth
  • Fixed: False "overwrite source" warning when transcoding to a different container format
